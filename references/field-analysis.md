## 核心原则

字段分析是"勘探"，需要看到数据原始面貌；Alpha回测是"测试"，需要用中和/衰减模拟真实交易环境。

**字段分析专用配置（MANDATORY）：**
- Neutralization = **None**（否则数据被中和后看不清特性）
- Decay = **0**（否则衰减后信号太弱）
- 其他设置：使用默认值

详见 [data-type-strategy.md](data-type-strategy.md) 的"基本面vs量价配置差异"

## 6种标准测试速查表

| # | 表达式 | 获取什么信息 | 判断规则 |
|---|--------|-------------|----------|
| 1 | `{field}` | 基础覆盖率 | Long+Short / 3000 ≈ 覆盖率% |
| 2 | `{field} != 0 ? 1 : 0` | 非零覆盖率 | Long Count = 有值的天数 |
| 3 | `ts_std_dev({field},N) != 0 ? 1 : 0` | 更新频率 | N=5周/22月/66季，看Long Count变化 |
| 4 | `abs({field}) > X` | 数据边界 | X=1判断是否归一化到[-1,1] |
| 5 | `ts_median({field},1000) > X` | 中位数分布 | 调X看百分位 |
| 6 | `scale_down({field})` | 归一化分布 | 看值域在[0,1]的分布 |

### 各测试详解

**Test 1 - 基础覆盖率**：直接用原始字段回测，Long Count + Short Count 之和反映该字段覆盖了多少股票。覆盖率低于30%的字段要慎用，补救方法见 [data-type-strategy.md](data-type-strategy.md) 的"低覆盖率字段补救"。

**Test 2 - 非零覆盖率**：过滤掉零值后看有多少股票有实际数据。和Test 1对比可以知道零值占比。

**Test 3 - 更新频率**：用标准差检测字段在N天内是否有变化。N=5检测周频，N=22检测月频，N=66检测季频。Long Count越小说明更新越慢（基本面数据特征）。

**Test 4 - 数据边界**：判断字段值是否在[-1,1]范围内。如果abs>1有值，说明未归一化，可能需要rank/scale处理。

**Test 5 - 中位数分布**：调整X值逐步查看数据的百分位分布，了解数据是否偏态。

**Test 6 - 归一化分布**：scale_down将数据压缩到[0,1]，观察分布是否均匀。

## 字段类型判断流程

```
拿到新字段名
  ↓
先跑 Test 1（基础覆盖）
  ↓
覆盖率 < 30%？
  ├── 是 → 低覆盖字段，参考 data-type-strategy.md 补救
  └── 否 → 继续

跑 Test 3（频率检测，N=22）
  ↓
Long Count 接近 Test 1 的50%？
  ├── 是 → 月度更新数据（基本面？）
  └── 否 → 可能日更新（量价？）

跑 Test 4（边界测试，X=1）
  ↓
有值？
  ├── 是 → 可能在[-1,1]范围
  └── 否 → 需要调整X再试
```

### 判断结果对照表

| 特征组合 | 字段类型 | 使用建议 |
|----------|----------|----------|
| 高覆盖 + 日更新 + 已归一化 | 量价类 | 适合做高频Alpha，Decay可设低 |
| 高覆盖 + 日更新 + 未归一化 | 量价类（原始值） | 需先rank/scale，再构建表达式 |
| 中覆盖 + 月更新 + 已归一化 | 基本面类 | Decay必须设高（10-20），换手率低 |
| 中覆盖 + 月更新 + 未归一化 | 基本面类（原始值） | 需先归一化，Decay设高 |
| 低覆盖 + 任意 | 低质量字段 | 慎用，参考 data-type-strategy.md 补救或放弃 |

## 字段分析工作流

```
用户提供字段名
  ↓
第一步：查本地 field_analyses 表
  - 调用 db.searchFieldAnalysis({field_name: keyword})
  - 如果找到 → 直接使用已有分析结论
  ↓
第二步：搜平台 API
  - 调用 searchFields({keyword})
  - 找到字段后用6种标准测试分析
  ↓
调用 analyzeField(fieldName) 执行分析
  - 自动生成6种测试表达式
  - 用空回测（Neutralization=None, Decay=0）执行
  - 汇总结果到结构化报告
  ↓
返回结构化数据：
  - 覆盖率: XX%
  - 更新频率: 日/周/月/季
  - 数据范围: [min, max]
  - 分布特征: 正态/偏态/集中
  ↓
询问用户是否保存到字段分析库
```

**关键区别**：字段分析结果存到 `field_analyses` 表，不污染 `alphas` 表。
