## 新论文加入流程

```
步骤1: 接收论文
  - 用户提供论文（PDF/链接/文本/摘要）
  - AI确认收到，告知将执行提炼

步骤2: 精读提炼（3要素）
  a) 核心发现（≤300字）
  b) 可复用策略模式（什么数据→什么逻辑→什么预期效果）
  c) BRAIN平台复现方式（对应数据集/字段/算子）

步骤3: 转化为平台可执行内容
  - 论文思路 → 表达式模板
  - 标注适用条件（数据类型/市场/频率）
  - 标注局限（什么时候不用）
  - 添加标签（#动量 #情绪 #基本面 等）

步骤4: 用户确认
  - 展示提炼结果，用户可修改/补充

步骤5: 写入知识库
  ├── 策略模式 → strategy-patterns.md 对应类别
  ├── 数据经验 → data-type-strategy.md
  ├── 论文索引 → strategy-patterns.md "论文启发索引"
  └── 状态标注：🧪待验证

步骤6: 建议验证
  - AI建议用哪些字段/模板验证
  - 验证成功 → ✅已验证
  - 验证失败 → ❌无效，记录原因

步骤7: 保存论文原文（可选）
  - 用户需要时保存到 papers/ 目录
  - 论文索引中链接到原文
```

**压缩格式标准**：
- 核心发现不超过300字
- 可复用模式必须链接到已有章节（保持索引一致性）
- 标签使用小写英文+下划线，如 #量价 #基本面 #情绪

## 新经验记录流程

```
实际测试中积累新经验
  ↓
判断经验类型
  ├── 数据集相关 → 追加到 data-type-strategy.md 的"数据集经验表"
  ├── 优化相关 → 追加到 optimization-guide.md 的"速查表"或"分类归因"
  ├── 策略相关 → 追加到 strategy-patterns.md 对应模式的"变异方向"
  └── 字段相关 → 保存到 field_analyses 表（非本目录文件）
  ↓
写入对应文件
```

**记录原则**：
- 只写平台特有知识，不写大模型已知的常识
- 经验必须来自实际测试，不记录未验证的推测
- 失败经验同样有价值，记录到optimization-guide.md

## 策略模式状态标注说明

| 标注 | 含义 | 判定标准 |
|------|------|----------|
| ✅已验证 | 在平台上回测通过，IS Checks全PASS | 至少1次成功提交记录 |
| 🧪待验证 | 逻辑合理但尚未在平台验证 | 从论文/课程提炼但未实测 |
| ❌无效 | 在平台验证失败，逻辑不可用 | 多次测试均无法通过IS Checks |

**状态流转**：
- 🧪待验证 → ✅已验证：首次成功提交后更新
- 🧪待验证 → ❌无效：3次以上验证失败后更新
- ✅已验证 → ❌无效：平台规则变更导致失效时更新

**注意**：状态标注是动态的，定期复查已验证模式是否仍然有效。

## 扩展路线图

### 阶段1（当前）：Markdown文件，AI直接读

- 论文数量：<10篇
- 知识库规模：<1000行
- 检索方式：AI全量读取references/目录
- 优势：简单直接，无需额外基础设施
- 限制：文件过大时AI上下文窗口吃紧

### 阶段2（论文>10篇）：拆分文件+索引

- 将strategy-patterns.md按策略类型拆分为独立文件
  - momentum-patterns.md
  - reversal-patterns.md
  - ...
- 增加index.md作为总索引
- 论文启发索引独立为paper-index.md
- AI先读索引，按需加载具体文件

### 阶段3（论文>50篇或知识库>1万行）：向量检索

- 引入SwarmVault MCP做向量检索
- 论文和经验转为向量嵌入
- AI通过语义搜索获取相关知识片段
- 保留Markdown文件作为人类可读备份
